# 풀업 체크 — 서비스 페이지 카피 가이드

> 톤: 전문적이지만 친근하게. 뻔한 표현 대신 구체적인 텍스트.
> 원칙: 기술 용어를 사용자 언어로, 행동의 대상을 명시, 불확실성 제거, 달성감 부여, 구체적 교정 가이드

---

## 1. 메인 화면 (Home)

| 위치 | 현재 | 제안 |
|------|------|------|
| 상단 레이블 | 오늘의 턱걸이 | 오늘의 턱걸이 |
| 앱 타이틀 | 풀업 체크 | 풀업 체크 |
| 점수 레이블 | 최근 종합 점수 | 최근 종합 점수 |
| 통계 항목 | 총 횟수 / 세트 / 평균 템포 | 총 횟수 / 세트 / 평균 템포 |
| CTA 메인 | 분석 시작 | **자세 분석하기** |
| CTA 서브 | 기록 보기 | **지난 기록 비교하기** |
| 하단 안내 | 모든 처리는 기기에서 이루어집니다. 영상 데이터는 서버로 전송되지 않습니다. | **영상은 기기에서만 처리됩니다. 서버 전송 없음.** |

### 변경 의도

- `분석 시작` → `자세 분석하기`: "무엇을" 분석하는지 명확히
- `기록 보기` → `지난 기록 비교하기`: 단순 열람이 아닌 "비교"라는 가치를 전달하여 재방문 유도
- 하단 안내: 같은 의미를 더 짧고 단정하게. 불필요한 수동태 제거

## 2. 분석 화면 (Analyze)

| 위치 | 현재 | 제안 |
|------|------|------|
| 페이지 타이틀 | 새 분석 | **분석 설정** |
| 안내문 | 촬영 각도와 입력 방식을 선택하세요 | **어떤 각도에서, 어떻게 촬영했나요?** |
| 섹션 타이틀 | 촬영 각도 | 촬영 각도 |
| 라디오: 정면 | 정면 | **정면 — 좌우 밸런스 분석** |
| 라디오: 후면 | 후면 | **후면 — 등 근육·견갑골 분석** |
| 버튼: 실시간 타이틀 | 실시간 카메라 | 실시간 카메라 |
| 버튼: 실시간 설명 | 실시간 측정 | **지금 바로 촬영하며 분석** |
| 버튼: 업로드 타이틀 | 영상 업로드 | 영상 업로드 |
| 버튼: 업로드 설명 | 녹화 영상 분석 | **이미 찍어둔 영상이 있다면** |

### 변경 의도

- 안내문을 질문형으로 바꿔 대화하는 느낌
- 라디오에 각도별 분석 차이를 서브텍스트로 추가하여 선택 이유 제공
- 업로드 서브텍스트: 사용자 상황을 묘사하여 진입 장벽 완화

### 실시간 카메라

| 위치 | 현재 | 제안 |
|------|------|------|
| 카메라 권한 거부 | 카메라 접근이 거부되었습니다. 권한을 허용해 주세요. | **카메라 권한이 필요합니다. 브라우저 설정에서 허용해 주세요.** |
| 시작 버튼 | 시작 | **측정 시작** |
| 정지 버튼 | 정지 | **일시정지** |
| 다음 세트 | 다음 세트 | 다음 세트 |
| 완료 버튼 | 완료 | **분석 완료** |

### 변경 의도

- "스켈레톤 분석"은 사용자에게 낯선 용어 → "관절 트래킹 오버레이"로 구체화
- "MediaPipe"는 내부 기술명 → 사용자 관점에서 "자세 인식 모델"로 번역
- 모델 다운로드 안내에 용량 명시하여 불확실성 제거
- `정지` → `일시정지`: "정지"는 종료처럼 느껴질 수 있음
- `완료` → `분석 완료`: 무엇이 완료되는지 명시
